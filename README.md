# 无人机拍卖算法（UAV Auction Algorithms）

---
## 一、什么是无人机拍卖算法
拍卖算法是一种任务分配算法，通过竞价的方式将任务分配给无人机。分配过程中，每个无人机将提交竞价，代表的是无人机评估自身状态和任务条件的一个综合分数。然后，任务会被分配给出价最高的无人机。通过这种方式，拍卖算法可以在保证任务分配效果的同时，减少计算复杂度。

## 二、核心工作流程

1. 任务发布：一个中心控制节点或某个发起机发布任务需求，在仿真算法中，以主程序为中心控制节点。

2. 出价：每架无人机根据自己的剩余电量、当前位置、任务距离及计算能力，计算出一个“成本”或“收益”，并据此提交报价。

3. 分配任务：系统根据规则选出最优的无人机。

## 三、常见的拍卖模型
| 模型类型 | 核心特点 | 典型适用场景 |
| :--- | :--- | :--- |
| **单物品拍卖** | 一次仅处理单个任务，算法逻辑简单，响应速度极快。 | 紧急单点侦察、临时突发任务。 |
| **组合拍卖 (Combinatorial)** | 允许无人机对“一组任务”进行打包竞价，考虑任务间的关联性。 | 快递路径优化（如顺路配送多个快件）。 |
|**单物品拍卖 (Single-Item)**|	一次仅处理单个任务，算法逻辑简单，响应速度极快。| 紧急单点侦察、临时突发任务。|
|**组合拍卖 (Combinatorial)**|	允许无人机对“一组任务”进行打包竞价，考虑任务间的关联性。|快递路径优化（如顺路配送多个快件）。|
|**序贯拍卖 (Sequential)**|	任务按顺序逐个拍卖，无人机会基于已中标的任务计划后续路径。	|电力线巡检：需按地理顺序依次访问多个杆塔。|
| **共识拍卖 (CBBA)** | 领域内最经典算法。通过机间通信达成一致，无需中心控制节点。 | 大规模集群协同、无基站/复杂电磁环境。 |
|**反向拍卖 (Reverse)**|	需求方发布任务，无人机通过降低“报价”（如报出更低能耗）来竞争。	|区域覆盖：在保证监测质量前提下寻找最低能耗方案。|
## 四、 深入应用案例分析

### 1. 基于 CBBA 的去中心化协同
**共识包出价算法** 是目前科研与工业界处理大规模集群任务分配的最常用方案。

* **双阶段机制**：
    1.  **出价阶段**：无人机在满足最大任务数 $L_T$ 的约束下，贪婪地将能产生最大边际收益的任务加入本地“任务包”。
    2.  **冲突解决阶段**：无人机通过机间通信（如代码中模拟的 `y_global_max` 同步）交换信息。若发现特定任务的出价低于他人（$z_{ij} \neq i$），则释放该任务及其后所有关联任务，确保全网逻辑一致性。
* **应用场景**：适用于大规模无人机集群在无基站、强干扰的动态环境下自主达成任务共识。

### 2. 考虑能耗平衡的动态拍卖
在长航时任务中，电量平衡（Load Balancing）是维持集群生存能力的关键。

* **核心策略**：在出价函数中引入剩余电量作为权重变量。

    $$Bid = \alpha \cdot \text{Energy\_Remaining} - \beta \cdot \Delta \text{Distance}$$
* **算法效果**：系统会自动规避让低电量无人机执行高耗能任务。这种机制能显著延长整个无人机群的整体工作寿命（Network Lifetime），防止部分节点过早因电量耗尽而离网。

### 3. 异步序贯拍卖
* **应用场景**：适用于任务动态产生的场景，如传感器实时触发的安防报警。
* **核心流程**：
    1.  无人机基于当前已规划的轨迹路径 $L$ 进行评估。
    2.  利用**插入启发式算法**计算加入新任务 $T_{new}$ 后的最小额外边际成本：
        $$\Delta C = Cost(L \cup \{T_{new}\}) - Cost(L)$$
* **算法优势**：支持任务的随时插入、撤销与重排，具有极高的动态响应速度，非常适合城市常态化安防巡逻任务。

## 五、核心算法实现

### 1. 序贯单项拍卖 (SSI - Sequential Single-Item Auction)
* **代码实现**: `SSI.py`, `BasicAuction.py`, `algorithms.py` 中的 `SSIAuction` 类。
* **核心逻辑**: 采用**插入启发式 (Insertion Heuristic)**。每一轮拍卖中，所有无人机尝试将剩余任务插入当前路径的每一个可能位置。
* **出价机制**: 计算插入新任务后的**边际成本（Marginal Cost）**，即新路径总长度与原路径长度之差。
* **特点**: 算法复杂度较低，通过贪婪策略在每一轮寻找全局最优的出价组合。

### 2. 带约束的 SSI 拍卖 (SSI with Constraints)
* **代码实现**: `ConstraintsSSI.py`
* **约束检查机制**: 在竞价前进行 `check_feasibility` 判定：
    * **载重约束 (Capacity)**: 无人机携带任务数量上限。
    * **航程约束 (Range)**: 模拟飞行能量限制，总里程不可超过最大阈值。
    * **时间窗 (Time Windows)**: 任务必须在指定的早到/晚到时间内开始执行。
* **处理逻辑**: 若任务违反约束导致路径无效，无人机将放弃报价；若所有飞机均无法满足约束，任务将被标记为“无法分配”。

### 3. 基于共识的捆绑拍卖 (CBBA - Consensus-Based Bundle Algorithm)
* **代码实现**: `CBBA.py`, `algorithms.py` 中的 `CBBA_Allocator` 类。
* **双阶段处理**:
    1. **任务包构建 (Bundle Construction)**: 无人机根据**时间折扣奖励模型**（$\lambda^{\tau}$）在本地构建最优任务序列。
    2. **冲突解决 (Conflict Resolution)**: 模拟分布式通信，通过对比全网最高出价列表 `y` 和胜出者列表 `z` 达成共识。
* **收敛性**: 发现任务被抢占后，无人机会释放该任务及其后续的所有任务，确保最终分配的一致性。


### 4. 组合拍卖 (Combinatorial Auction)
* **代码实现**: `CombinatorialAuction.py`
* **核心逻辑**: 解决**胜者决定问题 (WDP)**。通过 `itertools.product` 穷举所有可能的分配组合。
* **路径优化**: 对每一个分配方案，利用全排列暴力求解 **TSP (旅行商问题)** 以获取该组合下的最短路径。
* **特点**: 能够获得理论上的**全局最优解**，但计算量随任务数指数级增长，仅适用于极小规模集群。
